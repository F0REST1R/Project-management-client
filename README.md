Жизненный цикл Администратора (без отчетов):
Перед всей работой
- Создаю команду (программа выделит бд для нее)
- Добавляю всех участников команды, когда участники добавлены им не нужно регаться, а если они уже зареганы переключение между командами, если у него есть другие, если нет его автоматически привязывают к компанде, если он не админ, если админ предлагают перейти в команду
Приходит клиент с заказом
- Оформлаяем клиента, добавляем про него информацию + в какой-то момент добавляем ТЗ (Если есть)
- Создаем проект, указываем teamlead'а, бюджет, состав команды (либо это указывает тимлид) и всю остальную информацию

Жизненный цикл Руководителя команды (без отчетов):
После того как назначили руководителем проекта на почту приходит сообщение (Потом будет пункт оповещения)
- Набираю команду, смотрю кто занят, кто нет и их нагруженность, приблизительный расчет расходов на проект
- Создаю задачу и ставлю исполнителей
- Закрываю задачи, пишу комментарии, возвращаю задачи, закрываю проект (подтверждение админа)

Жизненный цикл Разработчика (Особо отчеты не нужны):
После того как меня назначают исполнителем
- Делаю свои задачи и ставлю статус, в общем dashboarde могу понаблюдать за всеми процессами, которые щас по проекту
- Пишу комментарии к задачам, завершаю их (с подтверждение руководителя)
